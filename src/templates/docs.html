WhiteRabbit Framework Documentation

Introduction

WhiteRabbit is a Python web framework built to balance simplicity and performance. Unlike large frameworks, WhiteRabbit focuses on essential web development features like routing, template rendering, static files, and middleware handling. It's ideal for small projects, learning, or building fast prototypes.

The framework is lightweight and requires minimal setup. It emphasizes readability, clean code structure, and reusability through components, so developers can scale their projects without being overwhelmed by unnecessary boilerplate.

Installation

git clone https://github.com/yourusername/whiterabbit.git
cd whiterabbit
pip install -r requirements.txt

Guide: Ensure Python 3.11+ is installed. Using a virtual environment is highly recommended: 

python -m venv venv
source venv/bin/activate  # Linux/macOS
venv\Scripts\activate     # Windows

This isolates your project dependencies and prevents conflicts with other Python packages.

Project Structure


whiterabbit/
    __init__.py        # Core framework initialization
    app.py             # Main app class
    router.py          # Handles route registration and dispatch
    response.py        # HTTP response utilities
    templates.py       # Template rendering engine
    middleware.py      # Middleware handling
src/
    app.py             # Application entry point
    routes/
        landing.py     # Example landing page routes
        docs.py        # Documentation routes
    components/
        navbar.py      # Reusable navigation bar component
    templates/
        index.html     # Homepage template
        docs.html      # Documentation template
    static/
        style.css      # Global CSS

Guide: Follow this structure for a maintainable project. - whiterabbit/ contains framework code. - src/ is where your application-specific code lives. - Keep templates organized and components reusable to maximize efficiency.

Getting Started

from whiterabbit import App
from src.routes import landing, docs

app = App()

if __name__ == "__main__":
    app.run()

Guide: This sets up the app instance and starts a local server. Navigate to http://127.0.0.1:8000 to see your first page. WhiteRabbit automatically discovers routes defined with @app.route decorators.

Routing

from whiterabbit import app
from whiterabbit.components.navbar import Navbar

@app.route("/")
def landing(request):
    navbar = Navbar()
    return app.render_template("index.html", {"navbar": navbar.render()})

Guide: Routes map URLs to functions. Each route receives a request object and should return HTML or other content. - Use parameters in paths: @app.route("/user/<username>"). - WhiteRabbit automatically parses these parameters and passes them to your function.

Templates

Use {{ variable }} in your HTML templates to insert dynamic data from your routes.

Guide: Templates live in src/templates/. Example: 

<h1>Hello, {{ username }}!</h1>

In your route: 

return app.render_template("index.html", {"username": "Robby"})

WhiteRabbit's template engine is lightweight and fast. It supports basic logic like loops and conditionals for dynamic page content.

Static Files

Place CSS, JS, images inside src/static/ and reference them using /static/filename.

Guide: - Organize files into subfolders like css/, js/, images/. - Link them in templates with: <link rel="stylesheet" href="/static/css/style.css">. - WhiteRabbit serves static files efficiently without extra configuration.

Components

Components are reusable pieces of HTML logic, such as navbars, footers, or custom widgets.

Guide: - Create classes in src/components/ with a render() method returning HTML strings. - Example: 

class Navbar:
    def render(self):
        return '<nav><a href="/">Home</a></nav>'

Render in a route: 

navbar = Navbar()
return app.render_template("index.html", {"navbar": navbar.render()})

Middleware

Middleware functions allow you to intercept requests and responses globally. Examples include authentication checks, logging, or error handling.

Guide: - Middleware functions accept request and optionally response. - Register with app.add_middleware(your_middleware_function). 

def logger(request, response=None):
    print(f"Request path: {request.path}")

app.add_middleware(logger)

Middleware is executed in the order registered, allowing flexible control over request/response lifecycle.

Running the App

python -m src.app
# Open http://127.0.0.1:8000

Guide: Run your application with Python. - Change port: app.run(port=8001). - Use debug=True for live reload and detailed error messages during development.

Troubleshooting & Common Problems

  • ModuleNotFoundError: Make sure all dependencies are installed using pip install -r requirements.txt.
  • Port Already in Use: Change the port in app.run(port=8001) or stop the conflicting process.
  • Templates Not Loading: Ensure templates are inside src/templates/ and names match exactly.
  • Static Files Not Loading: Verify src/static/ paths and correct references in templates.
  • Navbar/Components Not Showing: Check that component.render() is returned and passed to template context.
  • Server Fails to Start: Confirm Python version >=3.11 and virtual environment is active.
  • Debugging Tips: Use print() statements in routes or middleware to inspect data flow. Use browser dev tools for network/static file issues.
